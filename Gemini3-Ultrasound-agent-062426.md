智慧型醫療器材審查與法規合規自動化系統 (Agentic AI Regulatory Copilot)技術規格與系統架構設計說明書 (Technical Specification Document)本文件旨在定義一套專為智慧型醫療器材（如 Samsung Medison EVO Q10 診斷用超音波影像系統）設計的「多Agent協同醫療器材註冊申報與審查系統」（Agentic AI Regulatory Copilot）。本系統能自動解析使用說明書（IFU）、法規標準，自動提取技術特徵與探頭規格，執行 Gap Analysis（缺失分析），並根據法規導引與審查範本自動生成符合台灣 TFDA 與美國 FDA 510(k) 要求的審查報告與補件清單。1. 系統願景與多 Agent 核心架構 (System Vision & Multi-Agent Architecture)醫療器材的法規審查（Regulatory Submission & Review）是一項高度複雜、耗時且容錯率極低的專業工作。一份超音波系統的上市申報資料通常包含數千頁的技術文件、測試報告、軟體確效與 AI 演算法白皮書。傳統的人工審查極易遺漏細微的法規修訂或技術指標錯配。本系統採用基於大語言模型（LLM）的多智慧體協同架構（Multi-Agent Architecture）。將複雜的審查工作拆解為多個具備特定領域知識（Domain Knowledge）的虛擬專家（Agents），透過結構化的通訊協定、記憶體機制以及工具調用（Tool Calling），實現端到端的自動化醫療器材法規審查流程。[使用者介面: Streamlit / RAG 拖放區]

1.1 核心 Agent 角色定義與職責分工1.1.1 法規架構規劃專家 (Regulatory Schema Architect Agent)職責：主導審查報告的整體骨架與合規基線（Compliance Baseline）。辨識產品適用的法規代碼（如 IYN, IYO, ITX, LLZ）與引用標準（如 IEC 60601-1:2025、ISO 10993-1:2018）。輸入：使用者上傳的審查摘要、參考範本、申報國家法規指引。輸出：結構化的報告 JSON Schema 與審查清單檢驗矩陣。1.1.2 技術特徵提取專家 (Technical Feature Extractor Agent)職責：專職負責長文本（Long-context）解析。從數百頁的中文說明書（IFU）中，精確提取所有軟體功能、成像模式、硬體配置、探頭型號、晶體材質（如 Single Crystal 單晶體 vs PZT）、陣列元素、視野（FOV）、曲率半徑與對應的參考頁碼。輸入：原始 IFU 檔案（PDF/MD/TXT）。輸出：結構化、無幻覺（Hallucination-free）的軟體特徵與探頭規格數據庫。1.1.3 法規合規缺失專家 (Regulatory Gap Analyzer Agent)職責：比對技術特徵與法規標準之間的落差。對電性安全、電磁相容（EMC）、聲學輸出（IEC 60601-2-37）、生物相容性（ISO 10993-1）進行逐項稽核，判定「符合 [OK]」或「不符合 [缺失]」，並撰寫具體的補件要求（Deficiency List）。輸入：技術特徵數據、法規基線、評審筆記。輸出：Gap Analysis 矩陣與結構化缺失報告。1.1.4 臨床與 AI 驗證專家 (Clinical & AI Validation Expert Agent)職責：針對系統包含的 AI 醫療器材模組（如 CADe/CADx 軟體功能：ViewAssist, HeartAssist, S-Detect 等）進行深度技術審查。稽核演算法架構、數據集多樣性、獨立測試集劃分、人口統計學偏誤（BMI、人種）、黃金標準定義及統計學信心區間。輸入：AI 技術白皮書片段、說明書功能描述。輸出：AI 專項缺失分析與效能監控計畫審查意見。1.1.5 報告生成總編譯 (Report Compiler Agent)職責：聚合上述所有 Agent 的輸出結果，進行語意對齊、字詞精煉與排版優化。確保報告文字符合醫療器材審查員的專業術語規範，並將技術規格、缺失分析與補件清單完美融入使用者指定的報告範本中。輸入：各 Agent 分項 JSON 輸出、使用者提供的 Report Template。輸出：最終的繁體中文醫療器材審查報告（Markdown 格式）。2.
